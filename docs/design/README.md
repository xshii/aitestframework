## 4+1架构视图概述

本文档采用Philippe Kruchten的4+1视图模型描述系统架构：

```
                    ┌─────────────────┐
                    │   场景视图       │
                    │  (Scenarios)    │
                    │   用例驱动       │
                    └────────┬────────┘
                             │
         ┌───────────────────┼───────────────────┐
         │                   │                   │
         ▼                   ▼                   ▼
┌─────────────────┐ ┌─────────────────┐ ┌─────────────────┐
│   逻辑视图       │ │   进程视图       │ │   物理视图       │
│ (Logical View)  │ │ (Process View)  │ │(Physical View)  │
│   功能分解       │ │   运行时行为     │ │   部署拓扑       │
└────────┬────────┘ └─────────────────┘ └─────────────────┘
         │
         ▼
┌─────────────────┐
│   开发视图       │
│(Development View)│
│   代码组织       │
└─────────────────┘
```

| 视图 | 关注点 | 利益相关者 |
|------|--------|------------|
| 逻辑视图 | 功能需求、领域模型 | 用户、产品经理 |
| 开发视图 | 代码组织、模块划分 | 开发人员 |
| 进程视图 | 并发、性能、可用性 | 系统集成人员 |
| 物理视图 | 部署、拓扑、通信 | 运维人员 |
| 场景视图 | 用例、行为序列 | 所有人 |

# 1. 场景视图 (Scenarios)

场景视图通过核心用例驱动，贯穿其他四个视图。

## 1.1 核心用例

### UC-01: 执行测试用例

```
┌─────────────────────────────────────────────────────────────────────────┐
│  用例: 执行测试用例                                                       │
├─────────────────────────────────────────────────────────────────────────┤
│  参与者: 开发人员、CI系统                                                 │
│  前置条件: 框架已构建，平台HAL已配置                                       │
│  主流程:                                                                 │
│    1. 用户运行 test_runner                                               │
│    2. 框架初始化平台HAL                                                   │
│    3. 框架发现并加载所有测试用例                                           │
│    4. 按过滤条件筛选用例                                                  │
│    5. 逐个执行测试用例                                                    │
│       5.1 调用用例函数                                                    │
│       5.2 用例内执行断言                                                  │
│       5.3 收集执行结果                                                    │
│    6. 输出测试报告                                                        │
│  后置条件: 生成测试结果JSON/控制台输出                                     │
│  异常流程:                                                                │
│    E1: 用例超时 → 标记TIMEOUT，继续下一用例                                │
│    E2: 用例崩溃 → 信号恢复，标记CRASH，继续下一用例                         │
```

## 2.3 聚合模型

### TestCase聚合 (FWK)

```
TestCase (聚合根)
├── suite: STRING              # 套件名
├── name: STRING               # 用例名
├── func: TestFunc             # 函数指针
├── timeoutMs: UINT32          # 超时时间
├── tags: STRING               # 标签
└── result: TestResult         # 执行结果（值对象）
    ├── status: TestResultEnum
    ├── durationMs: UINT32
    ├── failFile: STRING
    └── failLine: INT32
```

### Platform聚合 (PLT)

```
Platform (聚合根)
├── name: STRING               # 平台名称
├── caps: PlatformCaps         # 能力（值对象）
│   ├── hasRealHardware: BOOL
│   ├── supportsDma: BOOL
│   └── supportsIrq: BOOL
└── hal: HalOps                # HAL操作集（实体）
    ├── Init/Deinit
    ├── RegRead/RegWrite
    └── MemAlloc/MemFree
```

### GoldenDataset聚合 (MDL)

```
GoldenDataset (聚合根)
├── name: STRING               # 数据集名称
├── version: STRING            # 版本
└── files: GoldenFile[]        # 文件列表（实体）
    ├── path: STRING
    └── header: GoldenHeader   # 值对象
        ├── dtype: Enum
        ├── ndim: UINT32
        └── shape[8]: UINT32
```

## 3.4 构建配置

```makefile
# Makefile结构
PLATFORM ?= linux_ut
CC = gcc
CFLAGS = -std=c99 -Wall -Wextra -Werror -I include
CFLAGS += -DPLATFORM_$(shell echo $(PLATFORM) | tr 'a-z' 'A-Z')

# 源文件分组
FRAMEWORK_SRCS = $(wildcard src/framework/*.c)
PLATFORM_SRCS = src/platform/platform.c
MODEL_SRCS = $(wildcard src/model/*.c)
TEST_SRCS = $(wildcard tests/**/*.c)

# 目标
all: build/bin/test_runner

build/bin/test_runner: $(FRAMEWORK_SRCS) $(PLATFORM_SRCS) $(MODEL_SRCS) $(TEST_SRCS)
	@mkdir -p build/bin
	$(CC) $(CFLAGS) -o $@ $^ -lm
```
